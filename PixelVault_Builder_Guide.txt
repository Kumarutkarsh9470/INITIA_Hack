BEFORE YOU BUILD ANYTHING — UNDERSTAND THE PRODUCT
================================================================================

PixelVault is a cross-game player economy on Initia MiniEVM.

The idea in one sentence: you mint an NFT that becomes your IDENTITY across
all games, and that NFT has its own WALLET that holds all your stuff.

Here's how it works, piece by piece:

  PlayerProfile — You mint one NFT per wallet. That NFT automatically gets
    its own wallet (a "Token Bound Account" / TBA) via ERC-6551. This TBA
    is where ALL your game stuff lives.

  TBA (Token Bound Account) — Your game wallet. It's controlled by your
    browser wallet (EOA), but items/tokens/badges go INTO the TBA, not the
    EOA. Think of the EOA as the key and the TBA as the safe.

  PXL Token — The base currency. Every game token trades against PXL on
    the DEX. Later, PXL becomes the chain's gas token.

  GameRegistry — A registry of all games. When registered, each game gets
    its own ERC20 token and ERC1155 item collection auto-deployed.

  PixelVaultDEX — A simple AMM (like mini Uniswap). Swap PXL ↔ game tokens.

  GasPaymaster — Players can pay gas in any game token. The paymaster swaps
    it to PXL behind the scenes. Players never need to hold gas tokens.

  AchievementBadge — Soulbound (non-transferable) ERC1155 badges. Games mint
    these for milestones. Badges build a reputation score.

  Marketplace — Players list/buy ERC1155 items. Buyers can pay in ANY game
    token — the marketplace auto-swaps via DEX.

  DungeonDrops — Demo game 1. Pay 10 DNGN, roll for loot (60% Common Sword,
    30% Rare Shield, 10% Legendary Crown). Items go to your TBA.

  HarvestField — Demo game 2. Stake HRV tokens, wait 100 blocks, harvest
    reward + seasonal item. Also goes to your TBA.


HOW EVERY GAME ACTION WORKS (read this twice, seriously)
=========================================================

Every single game action follows this path:

  Your browser wallet (EOA)
    → calls TBA.execute(gameContractAddress, 0, calldata, 0)
      → TBA forwards the call to the game contract
        → game contract does its thing (mints items, etc.)
          → items land in the TBA

Concrete example — when someone clicks "Enter Dungeon":

  1. Frontend encodes enterDungeon() into raw bytes (calldata)
  2. Frontend wraps that in: tba.execute(DungeonDrops address, 0, calldata, 0)
  3. Player's browser wallet signs the execute() call
  4. TBA checks "does the signer own the NFT?" → yes → forwards to DungeonDrops
  5. DungeonDrops charges 10 DNGN from the TBA, rolls dice, mints a loot item
  6. The loot item lands in the TBA (not the EOA)

This is why useTBA.ts exists. Every screen that does a game action calls:
  tba.execute(contractAddress, 0n, encodedCalldata)
...and useTBA handles the wrapping and signing internally.

The ONLY exception: PlayerProfile.mint() on the Create Profile screen is
called directly from the EOA (because the TBA doesn't exist yet).

--- Step 1: Make sure your appchain is running ---

  weave rollup start -d

  VERIFY IT WORKS:
  curl localhost:8545 -X POST -H "Content-Type: application/json" \
    -d '{"jsonrpc":"2.0","method":"eth_blockNumber","id":1}'

  ✅ You should see: {"jsonrpc":"2.0","id":1,"result":"0x..."} (some hex number)
  ❌ If you see "connection refused": the node isn't up yet. Wait 30 seconds, try again.


--- Step 2: Start the bridge bots ---

  weave opinit start executor -d
  weave relayer start -d


--- Step 3: Create .env file ---

In the pixelvault/ root (same folder as hardhat.config.ts), create a file
called .env with these three lines:

  MINIEVM_RPC_URL=http://localhost:8545
  PRIVATE_KEY=<your hex private key>
  CHAIN_ID=<your chain ID number>

How to get PRIVATE_KEY:
  initiad keys export gas-station --unsafe --unarmored-hex --keyring-backend test
  Copy the hex output. Add 0x at the front if it's missing.

How to get CHAIN_ID:
  curl localhost:8545 -X POST -H "Content-Type: application/json" \
    -d '{"jsonrpc":"2.0","method":"eth_chainId","id":1}'
  The result is hex. Convert to decimal (0x539 = 1337).

NEVER push .env to GitHub. It's already in .gitignore.


--- Step 4: Deploy ---

  cd pixelvault
  npx hardhat run scripts/deploy.ts --network minievm

  ✅ You should see every contract print its address, ending with "All contracts deployed"
  ❌ "insufficient funds": fund your gas station with INIT from the faucet
  ❌ "nonce too high": restart the node (weave rollup restart)


--- Step 5: Wire contracts ---

  npx hardhat run scripts/wire.ts --network minievm

  This grants permissions: which contracts can mint items, issue badges, etc.
  If you skip this, DungeonDrops and HarvestField won't work.

  ✅ You should see lines like "grantGameRole done", "grantIssuerRole done"


--- Step 6: Verify deployment works ---

    await pp.totalPlayers()
    // Should return 0n (BigInt zero) — no one has minted yet

  Type .exit to leave.

  git push

--- Understanding the hooks (you'll use these in every screen) ---

usePlayerProfile() — returns:
  {
    hasProfile: boolean,      // has the wallet minted a profile?
    tokenId: bigint,          // the NFT token ID
    tba: string,              // the TBA wallet address (0x...)
    username: string,         // player's display name
    reputation: bigint,       // reputation score from badges
    isLoading: boolean,       // true while fetching data
    refetch: () => void,      // call this after any action to refresh data
    mint: (username) => void, // ONLY used on CreateProfile screen
  }

useTBA() — returns:
  {
    execute: (target, value, calldata) => Promise<void>,
    // target = contract address to call
    // value = ETH to send (usually 0n)
    // calldata = encoded function call (from encodeFunctionData)
    isPending: boolean,       // true while tx is processing
  }

useAutoSign() — returns:
  {
    grantSession: () => void,   // opens the session approval popup
    isSessionActive: boolean,   // true if auto-signing is on
    revokeSession: () => void,  // ends the session
  }

--- SCREEN: HarvestField.tsx (pages/HarvestField.tsx) ---

WHAT IT IS:
  A staking game with 3 UI states. Player stakes HRV tokens, waits 100
  blocks, then harvests for a reward + seasonal item.

THE 3 UI STATES (show only ONE at a time):

  State 1 — Not staking (stakes.amount == 0):
    Show HRV balance, an amount input, and a "Stake" button.

  State 2 — Staking, waiting (amount > 0 but currentBlock < stakedAtBlock + 100):
    Show: staked amount, staked-at block, current block, progress bar
    (blocksElapsed / 100), estimated reward, disabled Harvest button,
    active Unstake button.
    Poll publicClient.getBlockNumber() every 5 seconds to update progress.

  State 3 — Ready to harvest (currentBlock >= stakedAtBlock + 100):
    Harvest button is enabled (green). Show final reward estimate.

WHAT CONTRACT FUNCTIONS YOU'LL CALL:
  Reads:
    - HarvestField.stakes(tba) → returns [amount, stakedAtBlock] as bigints
    - HRV token balanceOf(tba) → use ADDRESSES.HarvestFieldToken
    - publicClient.getBlockNumber() → current block number

  Reward formula (compute in JS for display):
    reward = (amount * 1e16 * blocksElapsed) / 1e18
    where blocksElapsed = currentBlock - stakedAtBlock

  Writes (all via useTBA().execute):
    - stake(uint256 amount)
    - harvest()        — no arguments
    - unstake()        — no arguments

  Swap PXL → DNGN:

    Step 2: do the swap
      calldata = encodeFunctionData({ abi: PixelVaultDEXABI, functionName: 'swapPXLForGame', args: [gameId, amount, minOut] })

DEBUGGING — READ THIS WHEN THINGS BREAK
================================================================================

Things WILL break. This section tells you what to do.

--- How to see errors ---

  Browser console: F12 (or Cmd+Option+J on Mac) → Console tab
  Terminal: look at the terminal where npm run dev is running

  The browser console error is usually the most useful.


--- The fix loop ---

  1. Copy the EXACT error message from the console
  2. Give it to your AI with context:
     "I'm getting this error in my DungeonDrops.tsx component:
      [PASTE ERROR]
      Here is the component code: [PASTE YOUR FILE]
      Here is useTBA.ts: [PASTE THE HOOK]
      Fix the error."
  3. Apply the fix, save, refresh the browser
  4. Repeat until it works


--- Common errors and what they mean ---

  "Cannot read properties of undefined (reading 'xxx')"
    → A hook returned undefined. Usually means usePlayerProfile() hasn't
    loaded yet. Add a check: if (!tba) return <Loading />

  "Contract function 'xxx' not found"
    → Wrong ABI imported. Check that you're using the right ABI for the
    right contract. DungeonDropsABI for DungeonDrops, not MarketplaceABI.

  "Execution reverted"
    → The contract rejected the call. Common reasons:
    - "Already staking" → player already has a stake, can't stake again
    - "No active stake" → trying to harvest/unstake without a stake
    - "Pool not active" → DEX pool hasn't been initialized with liquidity
    - "Insufficient balance" → TBA doesn't have enough tokens

  "Invalid signer"
    → The TBA is rejecting the call. This means the wallet calling execute()
    doesn't own the NFT. Check that you're connected with the right wallet.

  Blank page, no error
    → Usually a missing import or export. Check that your component is
    exported as default and that App.tsx imports it correctly.

  "Module not found: '@initia/interwovenkit-react'"
    → npm install wasn't run. cd frontend && npm install.


--- Common code mistakes (will save you hours) ---

  Token amounts have 18 decimals:
    1 DNGN = 1000000000000000000n = BigInt(1e18)
    If the user types "10" in an input, you need: parseEther("10")
    For display, use: formatEther(balance) → shows "10.0"

  gameId is bytes32, NOT a string:
    Don't pass "DungeonDrops" to a contract expecting bytes32.
    Use the pre-computed constants from lib/constants.ts:
      import { DUNGEON_GAME_ID, HARVEST_GAME_ID } from '../lib/constants'

  TBA address != PlayerProfile address:
    Items are at the TBA. Tokens are at the TBA. Don't use the profile
    contract address when checking balances.

  ownerToTokenId returns 0 for wallets that never minted:
    tokenId 0 IS valid (first player). Always check hasMinted() first.

  msg.sender vs _msgSender():
    Game contracts use _msgSender() because GasPaymaster does EIP-2771
    forwarding. You don't need to worry about this in the frontend. Just
    know it exists if you're reading contract code.

  Never call game contracts directly:
    Always go through useTBA().execute(). The TBA forwards the call.
    Exception: PlayerProfile.mint() is called directly by the EOA.

  PXLTokenInitia fee_denom format:
    'evm/<address>' — no 0x prefix, lowercase hex.
    Example: 'evm/5fbdb2315678afecb367f032d93f642f64180aa3'
